Release Overview

Development Activity Summary

This major release represents significant development activity with 195 commits over the recent development cycle. The av CLI continues to evolve as a comprehensive tool for managing stacked PRs on GitHub with enhanced functionality and improved user experience.

New Features

Major new functionality added in this release

Command Enhancements

  • Feature
    New Squash Command - Added comprehensive av squash command for consolidating commits within stacks, enabling better commit management and cleaner PR histories.
    # Squash commits in current branch av squash # Interactive squash with commit selection av squash --interactive
    PR #492 | Fix #575 | Tests #576
  • Feature
    Stack Split Enhancement - Added av stack split <new-branch> command to split the last commit into a new branch, providing more granular stack management capabilities.
    # Split the last commit into a new branch av stack split feature-branch-2 # This moves the last commit from current branch # to a new branch based on the parent
    PR #472
  • Feature
    Draft PR Support - Added --draft flag to av stack submit command, allowing users to open PRs as draft status for work-in-progress submissions.
    # Submit entire stack as draft PRs av stack submit --draft # Submit only current branch as draft av pr --draft
    PR #455
  • Feature
    Current Stack View - Added --current flag to av tree command to show only the current stack, improving focus and reducing visual clutter when working with multiple stacks.
    # Show all stacks (default) av tree # Show only the current stack av tree --current
    PR #580 | Fixes #550
  • Feature
    Enhanced Commit Amend - Added --all flag to av commit amend for more flexible commit modification workflows.
    PR #452
  • Feature
    Trunk Integration Enhancement - Added intelligent prompting when user is on trunk branch to suggest syncing all stacks instead of individual operations.
    PR #366
  • Feature
    Stack Navigation Enhancement - Improved stack prev command to allow moving to the trunk of the stack, enabling better stack navigation workflows.
    PR #317

Installation Guide

av CLI is available for macOS and Linux. For GitHub integration, install both av and GitHub CLI.

macOS (Homebrew)

brew install gh aviator-co/tap/av

Arch Linux (AUR)

yay av-cli

Debian/Ubuntu

echo "deb [trusted=yes] https://apt.fury.io/aviator/ /" | sudo tee /etc/apt/sources.list.d/fury.list sudo apt update && sudo apt install av

RPM-based Systems

sudo yum install av

Configure repository: /etc/yum.repos.d/fury.repo

Binary Download

Download from releases page

chmod +x av && sudo mv av /usr/local/bin/

Upgrade Instructions

How to upgrade from previous versions

Upgrade Process

Upgrading av CLI is straightforward and does not require any configuration changes.

macOS (Homebrew)

brew update brew upgrade av

Debian/Ubuntu

sudo apt update sudo apt upgrade av

RPM-based Systems

sudo yum update av

Post-Upgrade Verification

After upgrading, verify the installation:

av --version

Test basic functionality:

av tree
